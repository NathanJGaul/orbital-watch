# ADR-004: Redis pub/sub over Redis Streams

Capture: pub/sub chosen for simplicity and lower-latency delivery. Missed messages (if a subscriber is offline) are not recoverable. Streams provide persistence, consumer groups, and replay - required for audit logging or guaranteed delivery. For a live telemetry dashboard where stale frames are acceptable, pub/sub is the right trade. Revisit if the dashboard needs to replay the last N minutes on connect.
